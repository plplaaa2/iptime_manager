# 🛡️ ipTIME Manager 개발 프롬프트 & 주의 사항 가이드 (Developer Caution Guide)

이 문서는 **ipTIME JSON-RPC 통합 구성요소** 개발 및 유지보수를 수행하는 AI 코딩 어시스턴트가 **반드시 준수하고 기억해야 할 핵심 원칙과 기술적 주의 사항**을 정리한 가이드입니다. 다음 세션의 프롬프트나 시스템 지침에 이 내용을 그대로 주입하여 불필요한 시행착오를 예방하세요.

---

## 1. 윈도우 에디터 및 투명한 코드 수정 원칙 (CRITICAL)

### 🚨 에러 우회용 임시 스크립트 작성 금지
- 에디터 파일 치환 툴(`replace_file_content`, `multi_replace_file_content`)이 매칭 에러를 일으킨다고 해서, **수정용 임시 파이썬 스크립트를 작성하여 백그라운드에서 실행하는 등의 우회책을 절대 쓰지 마십시오.**
- 임시 스크립트를 써서 코드를 수정하면, **사용자가 에디터 상에서 변경된 코드의 Diff(차이점)를 인라인으로 확인하고 검토할 수 없어** 신뢰성이 무너집니다.

### 🛠️ 치환 에러의 근본적인 진단 및 해결
- 치환 에러의 주원인은 **윈도우 개행 문자(`\r\n` / CRLF)** 및 **인코딩(UTF-8)**, 혹은 눈에 보이지 않는 공백 문자의 불일치 때문입니다.
- 치환할 타겟 파일의 특정 구간을 `view_file` 등으로 정확하게 라인 단위 확인 및 분석한 다음, **줄바꿈과 공백 간격을 원본과 100% 동일하게 맞춰 치환 툴을 호출**하여 에러를 근본적으로 해결하십시오.
- 이를 통해 사용자가 에디터 상에서 실시간으로 코드가 어떻게 안전하게 갱신되었는지 시각적 Diff로 승인할 수 있도록 보장해야 합니다.

## 2. 프로젝트 핵심 비즈니스 가이드 및 안전 규칙

### 🚫 위험한 제어 스위치 배제 (공유기 재부팅 유발 방지)
- **IPTV 모드 변경(IGMP 외 공인 포트 지정 등)** 및 **인터넷 공유기능(NAT Mode, `nat/config`)** 설정은 공유기 하드웨어 칩셋의 즉각적인 강제 재부팅을 유반하여 가정 내 모든 Wi-Fi와 네트워크 연결을 중단시킵니다.
- **안전제일(Safety-First)** 기조에 입각하여, 이와 같이 네트워크 끊김 및 오작동 위험이 있는 제어 장치는 실물 엔티티(`switch.py`, `select.py`)에서 **완전히 제거 및 봉인 격리**합니다.
- 만약 나중을 위한 규격 참조가 필요하다면, 오직 **[web_api.md](file:///d:/HA/hacs/iptime_manager/custom_components/web_api.md)** 문서에만 정보성 텍스트로 보존하십시오.

### 💾 설정 유실 예방 및 사용자 커스텀 설정 보존 법칙
- 자동 재부팅 스케줄(`reboot/timer`), 나이트 LED 스케줄(`led/config`), QoS 대역폭 속도(`qos/config`)와 같은 가변 스케줄성 데이터를 변경할 때, 단순 스위치 토글 동작으로 인해 기존의 커스텀 시간이나 요일 정보가 공유기 초기화 값으로 유실되는 현상이 자주 발생합니다.
- 이를 방지하기 위해 `api.py` 인스턴스에 **사용자가 설정한 최신 값을 기억하는 보존 멤버 변수**(`_last_valid_hour`, `_last_valid_days`, `_last_valid_qos_up` 등)를 반드시 마련해 두십시오. 제어 명령 인가 시 이 값을 최우선 복구 결합하여 전송해야 설정 초기화 참사를 완벽히 예방할 수 있습니다.

### 🔒 GeoIP 국내 접속 허용 및 락아웃(Lock-out) 예방
- GeoIP 보안 정책을 특정 국가 허용(`accept`) 모드로 설정 시, 허용 리스트(`accept_list`)가 비어 있거나 사용자의 접속 대역인 한국(`kr`)이 누락되면 관리자의 외부 접속이 영구 차단되는 **락아웃 사고**가 발생합니다.
- 따라서 정책 설정 API(`geoip/policy/set`) 호출 전에 반드시 기존 리스트를 분석하여 **한국(`kr`)이 무조건 강제 병합 및 포함**되도록 안전 코드를 설계하십시오.

## 3. JSON-RPC (베타 UI) API 세부 규격 메모

1. **포트포워드 스위치 (`portforward/config`)**
   - 페이로드 형식: `{"enable": bool}`
2. **UPnP 포트포워드 릴레이 스위치 (`upnp/relay`)**
   - 페이로드 형식: 딕셔너리가 아닌 **단일 boolean 값 (`bool(enable)`)**을 다이렉트로 인자 전송해야 유효성 검사 에러를 예방할 수 있습니다.
3. **QoS 트래픽 차등화 (`qos/config`)**
   - 페이로드 형식: `{"mode": bool, "smart": bool, "up": str, "down": str}`
   - **대역폭 속도 문자열 접미사 필수:** 대역폭 설정값은 반드시 뒤에 **"Mbps"** 접미사가 붙은 형태(예: `"80Mbps"`)로 전달해야 공유기 하드웨어에서 오류 없이 수용됩니다.

## 4. 코드 컴파일 및 검증 절차

- 수정 및 치환 툴 적용이 끝나면 반드시 `py_compile` 모듈을 통하거나, CLI 터미널에서 구문 컴파일 테스트를 가동하여 **Syntax Error가 0개**임을 최종 확인하고 배포 단계를 밟아야 합니다.
- 한국어를 소스코드 주석에 적을 때는 반드시 파일 상단에 `# -*- coding: utf-8 -*-`을 보장하거나, 파일 인코딩이 깨지지 않도록 **UTF-8 표준 인코딩** 작성을 상시 점검하십시오.
