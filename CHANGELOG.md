# 📋 Changelog (변경 이력)

ipTIME Manager의 공식 버전별 핵심 변경 사항 및 개선 내역입니다.
본 프로젝트는 홈어시스턴트 HACS 배포 규격을 엄격하게 준수하며 사용자에게 가장 안정적이고 다채로운 공유기 제어 환경을 제공합니다.

## 🚀 [v1.0.2] - 2026-05-17 (스마트 DoS/보안 비활성화 실시간 감지 알림 배포)

이 버전은 ipTIME 공유기의 핵심 보안 및 DoS(서비스 거부) 방어 스위치가 꺼졌을 때, 사용자가 보안 위험을 인지하고 올바르게 대처할 수 있도록 보강한 스마트 보안 UX 강화 패치입니다.

### 🛡️ DoS/보안 제어 8종 실시간 비활성화 감지 및 한글 경고/설명 알림
* **실시간 비활성화 차단 감지**: 8종의 핵심 보안 스위치(CSRF, ARP Virus, SYN Flood, Smurf, IP Source Routing, IP Spoofing, Inbound/Outbound Ping) 중 켜져 있던 옵션이 꺼졌을 때 이를 실시간으로 추적 및 판단합니다.
* **지능형 사용자 알림 팝업**: 잘 모르는 일반 사용자나 가족이 실수로 핵심 보안 스위치를 꺼두는 것을 사전에 방지하기 위하여 홈어시스턴트 지속 알림(`persistent_notification`)을 자동으로 생성합니다.
* **친절한 한글 설명 및 위험성 안내**: 생성된 경고 알림창에는 **각 보안 기능이 구체적으로 어떤 위협을 막아주는지(친절한 한글 기능 개요)**와 함께 **비활성화 시 노출되는 잠재적 취약 위험성(🔴)**에 대한 정밀 명세를 포함하여 스마트홈의 자가 보안 대처 능력을 혁신적으로 높였습니다.
* **고위험 특수 조건 중복 알림 방지**: 원격 관리 포트가 개방되어 있는 취약 상태에서 CSRF 보호가 꺼지는 고위험 특수 상황에서는 기존의 중대한 보안 취약성 경고(`iptime_csrf_disabled_danger`)만 단독 생성되도록 중복 생성 억제 설계를 적용했습니다.

## 🚀 [v1.0.1] - 2026-05-17 (실물 공유기 현장 검증 및 세션/IPTV/WireGuard 핫픽스 릴리즈)

이 패치는 실물 EFM ipTIME 공유기(AX 시리즈)와의 실시간 라이브 연동 테스트 및 역공학 스니핑을 바탕으로 한 긴급 안정화 배포입니다. 인증 쿠키 유실로 인한 통신 단절을 원천 방지하고, WireGuard/IPTV/LED 스위치가 오작동하던 파라미터 유효성 검사 예외를 백퍼센트 완벽하게 진단 및 핫픽스했습니다.

### 🔒 EFM JSON-RPC 로그인 세션 쿠키 수동 병합 고도화
* **SameSite=Strict 제약 완벽 극복**: EFM ipTIME 공유기 펌웨어가 로그인(`session/login`) 성공 시 내려주는 `efm_session_id` 쿠키가 `SameSite=Strict` 옵션으로 인해 aiohttp ClientSession의 자동 `CookieJar`에 누락되는 치명적 문제를 포착했습니다.
* **수동 파싱 및 강제 주입 루틴 도입**: HTTP `Set-Cookie` 헤더를 직접 정규식 또는 세미콜론 기준으로 수동 파싱하여 추출한 뒤, 이후 매 API 요청마다 `Cookie` HTTP 헤더에 `efm_session_id=...` 형태로 수동 강제 주입하도록 로그인 프로세스를 고도화하여 세션 만료 및 Unauthenticated 에러를 원천 해결했습니다.

### 📺 IPTV 모드 제어 유효성 검사 통과 (Invalid params 예방)
* **포트 자동 필터링 수식 탑재**: 포트 지정이 필요 없는 사설(private - IGMP Proxy) 및 꺼짐(off) 모드 선택 시, `port` 매개변수가 Payload에 동반되어 전송되면 펌웨어 유효성 검사기에서 `Invalid params (-32602)` 에러를 내뱉는 현상을 발견했습니다.
* **동적 매개변수 다이어트**: 공인 IP 포트 지정이 필수인 모드(`public`, `macvlan`, `scs`)일 경우에만 포트 필드를 동반하고, 그 외에는 포트 필드를 완벽하게 걸러내어 전송하도록 제어 로직을 보강하여 에러 없는 완벽한 실시간 IPTV 전환 및 공유기 자체 자동 재부팅 반영을 성취했습니다.

### 🌐 WireGuard VPN 조회 및 조회 필드 맵핑 무결성 확보
* **실물 API Key/Value 트랜스폼(Transform)**: 실물 펌웨어의 `wg/server/show` API가 주는 활성화 여부 키가 `run`이 아닌 `active`, ip 주소가 `ip`가 아닌 `address`임과 `subnet` 필드가 생략되어 오는 구조적 차이를 완벽히 포착했습니다.
* **표준화 구조 변환기 이식**: 펌웨어가 내려주는 실물 데이터를 홈어시스턴트 규격(`run`, `ip`, `subnet`, `port`, `nat`)으로 자동 가공 및 매핑하여 수집 코디네이터에 무결하게 공급되도록 보강함으로써 센서 사용 불가 문제를 종식시켰습니다.

## 🚀 [v1.0.0] - 2026-05-17 (공식 정식 첫 릴리즈)

이 버전은 ipTIME 공유기 통합 구성요소의 기념비적인 첫 정식 메이저 출시 버전입니다. 기존 SNMP의 성능 한계를 극복하고 최신 AX 시리즈 공유기 및 베타 UI 환경까지 아우르는 초고성능 JSON-RPC (Web API) 전용 아키텍처로 완전히 탈바꿈했습니다.

### ⚠️ 중요: 브레이킹 체인지 (Breaking Changes)
* **엔티티 ID 중복 `iptime` 접두사 정리**:
  * 홈어시스턴트 기기명(`ipTIME AX3000Q` 등)과 개별 엔티티 명칭이 결합될 때 `iptime`이 중복 생성되는 현상(`binary_sensor.iptime_ax3000q_iptime_lan_port_3_status`)을 수정했습니다.
  * 모든 엔티티의 이름에서 불필요한 `ipTIME ` 접두사를 제거하여 더 정갈한 ID(예: `binary_sensor.iptime_ax3000q_lan_port_3_status`)로 생성됩니다.
  * **기존 사용자 주의**: 통합 구성요소 업데이트 후 기존에 설정해둔 **러브레이스 대시보드 카드** 및 **자동화 스크립트(Automation)**의 엔티티 ID가 변경될 수 있으므로, 새로운 깔끔한 ID로 대상을 한 번 갱신해 주시기 바랍니다.

### 💻 모니터링 및 성능 모니터링 고도화
* **JSON-RPC 웹 API 기반 수집 일원화**: 복잡한 SNMP 설정 및 추가 모듈 활성화 요구를 완벽히 제거하고 오직 관리자 계정 정보만으로 모든 상태를 동기화합니다.
* **코어 시스템 텔레메트리**: 공유기 가동 시간(Uptime), 공유기 모델명 정규화(대문자), 펌웨어 버전(최신 버전 실시간 비교 비교), WAN IP 및 MAC 주소, Primary/Secondary DNS 센서를 노출합니다.
* **네트워크 포트 실시간 시각화**: WAN 및 LAN 1~4 포트의 연결 상태를 `connectivity` 디바이스 클래스의 이진 센서(`binary_sensor`)로 구현하였으며, 연결 여부에 따라 `mdi:ethernet` 및 `mdi:ethernet-off` 아이콘이 유기적으로 토글됩니다.

### 🛡️ 신규 제어 스위치 및 셀렉터 대거 추가
* **IPTV 설정 원클릭 변경**: 공식 매뉴얼에서는 불가능했던 SKB, LGU+, KT(MACVLAN 포함), 서경방송(SCS - 공인/사설) 등 6종의 특수 IPTV 모드를 지원하는 셀렉터(`select.iptime_iptv_mode`) 엔티티를 탑재했습니다.
* **WireGuard VPN 제어**: 외부 접속용 WireGuard VPN 서버의 기동 상태 모니터링 및 실시간 토글 제어 스위치(`switch.iptime_wireguard_server`)를 연동했습니다.
* **AX 시리즈 보안 8종 스위치**: 원격 관리, CSRF, ARP Virus, Inbound Ping 등 8종의 보안 제어 스위치를 개별 엔티티로 완벽 제공합니다.
* **GeoIP 보안 정책 제어**: GeoIP 정책 설정 셀렉터 및 GeoIP 차단 건수 실시간 누적 건수 감시 센서를 구현했습니다.
* **Wi-Fi BSS(SSID) 개별 제어**: Wi-Fi 대역(2.4G / 5G / 6G) 제어 시 IoT 기기의 순단을 예방하고 전체 무선이 중단되지 않도록 SSID 단위 개별 토글 처리를 적용했습니다.

### 👥 스마트한 재실 감지 (Presence Detection) 및 UX 개선
* **무파괴 타깃 분리 저장 아키텍처**: 사용자가 추적 대상을 0개로 비워두고 옵션을 저장하더라도 기존 등록 기기 데이터(`devices`)가 유실되지 않도록 보존 메커니즘을 심어 안정성을 높였습니다.
* **MAC 주소 시각화 표준화**: 기기 추가/설정 변경 화면의 MAC 주소 표현 방식을 대문자와 콜론 구분자 표준 규격(`00:11:22:33:44:55`)으로 정규화하여 시인성을 높였습니다.
* **글로벌 표준 다국어(i18n) 수립**: 영어(`en.json`)와 한국어(`ko.json`)의 고품격 번역을 완비하여 현지화 품질을 높였습니다.
